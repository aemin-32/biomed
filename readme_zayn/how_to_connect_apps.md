# الدليل الشامل لربط البرنامجين سحابياً ومجاناً (Cloud Integration Guide)

هذا الملف يشرح كيفية ربط **برنامج الهندسة الطبية (BioMed OS)** مع **برنامج الإدارة العليا (Directors Portal)** بحيث يقرآن ويكتبان على نفس قاعدة البيانات، مع استضافة مجانية على GitHub.

## 1. الفكرة المعمارية (Architecture)
بدلاً من استخدام ملفات محلية (`mockData.ts`)، سنقوم بنقل البيانات إلى **قاعدة بيانات سحابية (Cloud Database)**.
- **التطبيق الأول (BioMed OS):** يقرأ ويكتب (يضيف أجهزة، يطلب قطع، يسجل صيانة).
- **التطبيق الثاني (Directors Portal):** يقرأ بشكل أساسي (يعرض إحصائيات، تقارير، ويوافق على طلبات).
- كلاهما يتصلان بنفس قاعدة البيانات عبر الإنترنت (API).

## 2. أفضل الخيارات المجانية لقواعد البيانات (BaaS)
بما أنك تبحث عن حلول مجانية وموثوقة من شركات كبرى حتى تبني سيرفراتك الخاصة، إليك أفضل خيارين:

### الخيار الأول: Firebase (من Google) - الموصى به
- **المميزات:** مجاني بالكامل للبدايات (خطة Spark)، سهل جداً في الربط مع React، يوفر قاعدة بيانات NoSQL (Firestore)، ويوفر نظام تسجيل دخول (Authentication) جاهز.
- **كيفية الربط:**
  1. إنشاء مشروع على [Firebase Console](https://console.firebase.google.com/).
  2. تفعيل Firestore Database و Authentication.
  3. أخذ مفاتيح الربط (API Keys) ووضعها في ملف `.env` في كلا البرنامجين.
  4. استبدال `mockData.ts` بدوال تجلب البيانات من Firestore.

### الخيار الثاني: Supabase (البديل المفتوح المصدر لـ Firebase)
- **المميزات:** مبني على قاعدة بيانات علائقية قوية جداً (PostgreSQL)، خطة مجانية ممتازة، رائع جداً للبيانات المترابطة (مثل ربط الجهاز بالمهندس وبسجل الصيانة).
- **كيفية الربط:**
  1. إنشاء مشروع على [Supabase](https://supabase.com/).
  2. إنشاء الجداول (Devices, Users, Logs, Inventory).
  3. استخدام مكتبة `@supabase/supabase-js` في كلا البرنامجين للاتصال بقاعدة البيانات.

## 3. خطة الاستضافة (Hosting Strategy)
بما أنك سترفع الكود على **GitHub**، فهذه هي الطريقة الأفضل للاستضافة المجانية:

1. **Vercel أو Netlify:** 
   - قم بربط حسابك في GitHub بمنصة [Vercel](https://vercel.com/) أو [Netlify](https://netlify.com/).
   - بمجرد رفع أي تعديل (Push) على GitHub، ستقوم المنصة بتحديث الموقع تلقائياً.
   - ستحصل على رابطين منفصلين مجانيين:
     - `biomed-os.vercel.app` (للمهندسين)
     - `hospital-directors.vercel.app` (للمدراء)

2. **GitHub Pages:**
   - خيار آخر مجاني بالكامل من GitHub نفسه، لكن Vercel أسهل وأسرع لتطبيقات React (Vite).

## 4. خطوات التنفيذ العملي (Roadmap)
1. **الخطوة الأولى:** إكمال واجهات وتصميم برنامج المهندسين (BioMed OS) بالكامل باستخدام البيانات الوهمية الحالية.
2. **الخطوة الثانية:** إنشاء مستودع (Repository) جديد على GitHub لبرنامج المدراء وبناء واجهاته.
3. **الخطوة الثالثة:** إنشاء حساب Firebase أو Supabase ونقل البيانات الوهمية إليه.
4. **الخطوة الرابعة:** ربط كلا البرنامجين بقاعدة البيانات السحابية.
5. **الخطوة الخامسة:** رفع البرنامجين على Vercel للحصول على روابط حية ومجانية.
